## 2024-06-15 - [CRITICAL] Information Disclosure in Asynchronous Background Processes
**Vulnerability:** The clipboard sharing script dumped sensitive clipboard contents (e.g., passwords, keys) to a temporary file in `/tmp/` and passed it to a background process (`systemd-run`). The script exited immediately without deleting the file, permanently leaving sensitive data on disk.
**Learning:** Asynchronous background tasks (like `systemd-run` without `--wait` or disowned processes) that operate on temporary files prevent the parent script from cleaning up those files before exiting. A simple `rm -f "$TMP"` in the parent or using a trap on `EXIT` is insufficient because the background process might still be reading it.
**Prevention:** Always place sensitive temporary files in an in-memory tmpfs (e.g., `${XDG_RUNTIME_DIR:-/tmp}`). Wrap the asynchronous background execution in a shell command that handles the deletion *after* the command completes (e.g., `systemd-run ... sh -c 'command "$1"; rm -f "$1"' _ "$TMP"`).
