## 2024-05-15 - [CRITICAL] Fix Clipboard Data Leakage in Sharing Script
**Vulnerability:** Sensitive clipboard data could persist on disk indefinitely or until reboot. The clipboard contents were written to `/tmp/omanix-share-XXXXXX.txt` and passed to a background service via `systemd-run`. Since `systemd-run` launches the process asynchronously, the script exits immediately and no proper cleanup is done for the file.
**Learning:** `systemd-run` runs asynchronously, meaning a file passed to it won't be deleted automatically if the caller process relies on basic cleanup mechanisms or exits before the background process consumes the file. If an `EXIT` trap was used, it would prematurely delete the file before the background service could read it. Additionally, writing to `/tmp` might persist the sensitive content on physical disk depending on system configuration, instead of storing it purely in memory.
**Prevention:**
1. Use `${XDG_RUNTIME_DIR:-/tmp}` for temporary files handling sensitive data to ensure they reside on a memory-backed file system (tmpfs), guaranteeing they are wiped on reboot and do not hit physical storage.
2. When passing temporary files to asynchronous processes (like `systemd-run`), rely on the asynchronous command itself to perform the cleanup by wrapping the execution in a shell command (e.g., `sh -c 'command "$1"; rm -f "$1"' _ "$FILE"`).
