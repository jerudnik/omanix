## 2026-06-26 - Secure Temporary File Handling in Async Processes
**Vulnerability:** Clipboard content (which may contain passwords or API keys) was written to disk (`/tmp`) and passed to an asynchronous `systemd-run` process without any cleanup. If the content was sensitive, it could remain permanently on the disk, or be intercepted by other processes due to lack of strict file permissions.
**Learning:** `mktemp` in `/tmp` by default writes to a persistent disk layout depending on the OS, and asynchronous background processes do not automatically clean up their arguments. Furthermore, any cleanup trap inside the main script fires immediately upon script exit, which occurs before the asynchronous process finishes executing, prematurely deleting the file or leaving it intact if no trap is defined.
**Prevention:** Always use `${XDG_RUNTIME_DIR:-/tmp}` to ensure temporary files are written to tmpfs (in-memory) when available. Restrict file permissions using `chmod 600`. When passing temporary files to asynchronous processes (e.g. `systemd-run`), wrap the execution in a shell (`bash -c 'command "$1"; rm -f "$1"' _ "$FILE"`) to guarantee secure cleanup only *after* the command completes.
