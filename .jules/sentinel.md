## 2024-05-24 - Leaking Sensitive Clipboard Data to /tmp

**Vulnerability:** The clipboard sharing script (`omanix-cmd-share.sh`) was dumping the entire clipboard contents (which may contain passwords, API keys, or private messages) into a temporary file in the world-readable `/tmp` directory. Furthermore, when delegating the sending to a background service via `systemd-run`, the script failed to delete the temporary file after successful execution, leaving sensitive clipboard data lingering indefinitely on the filesystem.
**Learning:** Background services initiated with `systemd-run` (without `--wait`) return immediately, causing the parent script to exit. Therefore, trailing `rm` commands in the script do not run *after* the background task finishes, resulting in orphaned temporary files containing sensitive data. Additionally, using `/tmp` is less secure than utilizing memory-backed tmpfs partitions for temporary secrets.
**Prevention:** Always use `${XDG_RUNTIME_DIR:-/tmp}` for temporary files to leverage in-memory tmpfs partitions. When passing a temporary file to an asynchronous background task, wrap the background command in a shell execution (e.g., `bash -c 'command "$1"; rm -f "$1"' _ "$FILE"`) to ensure the file is securely deleted *after* the background task actually completes.
